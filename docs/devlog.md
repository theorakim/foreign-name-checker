출판사 편집자가 쓸 수 있는 외래어 표기법 검사기를 만들었다. 텍스트를 통째로 넣으면 외래어 후보를 자동 추출하고, 국립국어원 용례 API로 맞춤법을 확인해주는 도구.

## 배경

지인의 지인이 출판사(반비, 민음사 자회사) 편집자인데, 번역서 작업할 때 외래어 표기법 확인이 번거롭다고 했다. 인명, 지명 같은 고유명사를 하나하나 국립국어원 사이트에서 검색하는 게 시간이 많이 걸린다는 것. 문단을 통으로 넣으면 알아서 외래어를 골라내고 교정 결과를 알려주는 도구가 있으면 좋겠다는 요청이었다.

## 핵심 흐름

텍스트 입력 → 외래어 후보 자동 추출 → 사람이 확인/수정 → API 검색 → 결과 표시

중간에 사람이 확인하는 단계를 넣은 게 핵심이다. 처음에는 형태소 분석기로 명사를 전부 뽑아서 API에 다 보내는 방식으로 했는데, 175개 단어 중 외래어가 10개밖에 안 되는 상황이 나왔다. API 호출이 너무 낭비되니까, 먼저 추출 → 사람이 확인 → 그다음 API 호출 순서로 바꿨다.

## 기술 스택

- Python Flask (백엔드, 로컬 서버)
- kiwipiepy (한국어 형태소 분석기)
- 국립국어원 어문 규범 용례 찾기 API (공공데이터포털에서 키 발급)
- 바닐라 HTML/CSS/JS (프론트엔드)

## 외래어 후보 추출 전략

세 가지 전략을 조합해서 쓴다.

**1. 괄호 패턴**

번역서에서 흔한 "레오노르 안투네스(Leonor Antunes)" 형태를 정규식으로 잡는다. 앞에 "아티스트", "감독" 같은 일반어가 붙으면 자동으로 떼어낸다.

**2. kiwipiepy NNP 태그**

형태소 분석기가 고유명사(NNP)로 태그한 토큰을 추출한다. 연속된 NNP 토큰은 병합해서 "주디스 스콧" 같은 이름을 하나로 합친다. 일반명사(NNG) 중에서도 외래어 패턴이 있으면 후보에 넣는다. "오브제", "핸드폰" 같은 단어가 이렇게 잡힌다.

**3. 조사 제거 + 외래어 패턴 휴리스틱**

"비트겐슈타인의" → "비트겐슈타인"처럼 조사를 떼어낸 다음, 외래어에서 흔한 패턴(받침 크/트/프/스, 격음 카/타/파 등)이 있는지 체크한다.

## 외래어 패턴 판별

`_has_foreign_pattern()` 함수가 단어의 외래어 여부를 휴리스틱으로 판단한다.

- 외래어 특유의 받침으로 끝나는지 (크, 트, 프, 스, 즈, 드, 그, 브, 흐)
- 격음/경음 음절이 포함됐는지 (카, 타, 파, 키, 티, 피 등)
- 외래어 특유 음절 조합이 2개 이상인지 (핸, 폰, 브, 젝, 션 등)
- 특정 패턴 포함 여부 (르, 슈, 츠, 오브)

false positive를 줄이기 위해 `_COMMON_KOREAN` 세트에 "포함", "토론", "투자" 같은 한국어 단어를 넣어서 걸러낸다.

## API 검색 로직

국립국어원 API는 두 단계로 검색한다.

1단계는 정확히 일치(equal) 검색. "안투네스"가 그대로 등록돼 있으면 "올바른 표기"로 판정.

2단계는 부분 일치(like) 검색. 정확히 일치하는 게 없으면 유사한 용례를 찾아서 "확인 필요"로 표시하고 권장 표기를 제안한다. 유사도 판별은 포함 관계, 접두어 일치, 글자 겹침 비율로 한다.

중국어/일본어 한자 음역은 별도로 필터링한다. "안다"가 安達로 매칭되는 식의 false positive를 방지하기 위해.

## 삽질 기록

**kiwipiepy가 외래어 이름을 쪼갠다.** "보렌슈타인"을 "보"+"렌"+"슈타인"으로 분리한다. 연속 NNP 토큰을 원문 위치 기반으로 병합하는 로직으로 해결했다.

**NNP만으로는 부족하다.** "오브제"나 "핸드폰"은 kiwipiepy가 일반명사(NNG)로 태그한다. 고유명사만 추출하면 이런 일반 외래어가 빠진다. NNG 중에서도 외래어 패턴이 있으면 후보에 넣도록 확장했다.

**괄호 패턴이 너무 넓게 잡힌다.** "포르투칼 아티스트 레오노르 안투네스(Leonor Antunes)"에서 "포르투칼 아티스트"까지 같이 잡혔다. 접두어 일반어 제거 함수(`_trim_prefix`)를 추가하고, 중복 제거 로직(짧은 후보가 긴 후보에 포함되면 긴 쪽 삭제)을 넣었다.

**API 호출 낭비.** 175개 단어를 전부 API에 보내서 10개만 결과가 나온 적이 있다. 자동 추출 → 사람 확인 → API 호출 순서로 플로우를 바꿔서 해결.

## 파일 구조

```
foreign-name-checker/
  app.py              # Flask 백엔드 (추출 + API 검색 + 라우트)
  checker.py           # CLI 버전 (초기 프로토타입)
  templates/
    index.html         # 웹 UI (아틀리에/테라코타 톤)
  .env                 # API 키 (gitignore)
  .gitignore
  requirements.txt
```

## 실행 방법

```bash
cd foreign-name-checker
pip install flask kiwipiepy requests python-dotenv
python app.py
# http://127.0.0.1:5001 접속
```

`.env` 파일에 공공데이터포털에서 발급받은 API 키 필요.

```
KORNORMS_API_KEY=발급받은키
```

## 한계와 개선 여지

- kiwipiepy가 잘못 쪼갠 조각("비엔나", "포르투")이 후보에 섞여 나온다. 사람이 체크 해제하면 되지만 깔끔하진 않다.
- `_COMMON_KOREAN` 세트를 수동으로 관리해야 한다. 새로운 false positive가 나올 때마다 추가해야 함.
- 현재 로컬 전용. 나중에 웹앱으로 배포하는 것도 가능.
- 국립국어원 API에 등록되지 않은 외래어는 "용례 없음"으로 나온다. API 커버리지 자체의 한계.
